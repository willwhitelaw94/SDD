# Trilogy QA Test Codify

Converts **passing browser-based QA evidence** into **deterministic Playwright E2E test files**. Takes the test report from `/trilogy-qa-test-agent` and turns each verified acceptance criterion into a repeatable, automated browser test that runs in CI.

## Separation of Concerns

| Skill | Responsibility |
|-------|---------------|
| `/trilogy-qa` | **Plans** — generates qa-plan.md from spec ACs (no browser) |
| `/trilogy-qa-test-agent` | **Executes** — opens browser, walks through test plan, fixes failures, updates report |
| `/trilogy-qa-test-codify` | **Codifies** — THIS SKILL: converts passing browser tests → Playwright E2E |

## When to Use

```bash
/trilogy-qa-test-codify                    # Convert all passing ACs to Playwright tests
/trilogy-qa-test-codify --report path      # Point to specific test report
/trilogy-qa-test-codify --ac 1,3,5         # Only codify specific ACs
```

**Run this when:**
- All ACs pass (after `/trilogy-qa-test-agent`)
- You want browser tests to be permanent and run in CI
- You're preparing for Gate 5 handover and want regression coverage

## Prerequisites

| Artifact | Required | Source |
|----------|----------|--------|
| test-report.md | Yes | Generated by `/trilogy-qa-test-agent` — contains passing ACs with evidence |
| qa-plan.md | Recommended | Generated by `/trilogy-qa` — E2E test specifications section |
| spec.md | Yes | Initiative folder — full AC definitions |
| Existing Playwright tests | Reference | `e2e/tests/` — follow existing patterns |

## Running Playwright Tests

**Always run in headed mode with `E2E_LOCAL_DEPLOYMENT=true`:**

```bash
# Run a feature's tests (headed)
cd e2e && E2E_LOCAL_DEPLOYMENT=true npx playwright test tests/{feature}/ --headed

# Run a specific test file
cd e2e && E2E_LOCAL_DEPLOYMENT=true npx playwright test tests/{feature}/{test}.spec.ts --headed

# Run with slow-motion for debugging (1 second between actions)
cd e2e && E2E_LOCAL_DEPLOYMENT=true npx playwright test tests/{feature}/ --headed --slow-mo=1000
```

**Critical:** Without `E2E_LOCAL_DEPLOYMENT=true`, the `baseURL` won't resolve from `.env` and all tests fail with "Cannot navigate to invalid URL".

### Slowing Down Tests

The Playwright config has a `slowMo` option in `launchOptions` (line ~90 of `playwright.config.ts`) that is commented out by default. To slow down:

1. **Per-run** (recommended): Pass `--slow-mo=1000` flag when running headed
2. **Globally**: Uncomment `slowMo: 2000` in `playwright.config.ts` (remember to re-comment after debugging)

Use slow-mo when the user wants to visually follow along or when debugging flaky step transitions.

## Execution Flow

### Phase 0 — Pre-Flight Database Check

Before running any Playwright tests, ensure the database has seeded data:

```bash
# Check if users exist (Playwright tests need loginable users)
php artisan tinker --execute="echo \Domain\User\Models\User::count();"
```

If 0 users or the query fails:
```bash
# Try snapshot restore first (~2s)
php artisan db:snapshot restore --no-interaction

# If no snapshot exists, full seed + save snapshot for next time
php artisan migrate:fresh --seed --no-interaction
php artisan db:snapshot save --no-interaction
```

All test users use password `foobar`. Key accounts:
- `admin@example.com` (Admin)
- `care_partner@example.com` (Care Partner)
- `supplier.verified@example.com` (Supplier Administrator)

### Phase 1 — Gather Context

1. **Read the test report** — extract all PASS ACs with their:
   - Criteria description
   - Browser flow steps (from the evidence/screenshots)
   - URL paths tested
   - Form data used
   - Expected outcomes

2. **Read spec.md** — get full Given/When/Then scenarios for each AC

3. **Read existing Playwright tests** for patterns:
   ```bash
   ls e2e/tests/
   ls e2e/pages/
   ```
   - Check existing Page Objects and specs as reference
   - Note conventions: `l_` prefix for locators, `f_` prefix for functions

4. **Identify test data needs** — what seeded data is needed for each AC

### Phase 2 — Create Page Object

5. **Create Page Object** in `e2e/pages/{feature}.page.ts`:

   ```typescript
   import { expect, Locator, Page } from "@playwright/test";

   export class FeatureName {
       /** Section description */
       l_some_button: Locator;
       l_some_input: Locator;

       constructor(private page: Page) {
           this.l_some_button = page.getByRole('button', { name: 'Button Text' });
           this.l_some_input = page.getByPlaceholder('Placeholder text');
       }

       /** Navigate to feature index */
       async f_goto_index() {
           await this.page.goto('/feature');
           await this.page.waitForLoadState('networkidle');
       }

       /** Navigate to create page */
       async f_goto_create() {
           await this.page.goto('/feature/create');
           await this.page.waitForLoadState('networkidle');
       }

       /** Fill and submit the form */
       async f_create_item(options: { name: string; description: string }) {
           await this.l_some_input.fill(options.name);
           // ... more interactions
       }

       /** Assert we landed on the show page */
       async f_assert_on_show_page() {
           await expect(this.page).toHaveURL(/\/feature\/\d+/);
       }

       /** Assert validation error is shown */
       async f_assert_validation_error(errorText: string) {
           await expect(this.page.getByText(errorText)).toBeVisible();
       }
   }
   ```

6. **Register in barrel export** — add to `e2e/pages/index-pages.ts`:
   ```typescript
   export { FeatureName } from './feature-name.page';
   ```

### Phase 3 — Create Test Specs

7. **Create spec file** in `e2e/tests/{feature}/{test-name}.spec.ts`:

   ```typescript
   import { test, expect } from "@playwright/test";
   import { Auth } from '../../pages/index-pages';
   import { FeatureName } from '../../pages/index-pages';

   test.describe('Feature Name', () => {
       let auth: Auth;
       let feature: FeatureName;

       test.beforeEach(async ({ page }) => {
           auth = new Auth(page);
           feature = new FeatureName(page);
           await auth.f_log_in_as('admin');
       });

       test('can create an item through the form', async () => {
           await feature.f_goto_create();
           await feature.f_create_item({
               name: 'Test Item',
               description: 'A test description',
           });
           await feature.f_assert_on_show_page();
       });

       test('validation blocks submit without required fields', async () => {
           await feature.f_goto_create();
           await feature.l_some_button.click();
           await feature.f_assert_validation_error('Name is required.');
       });
   });
   ```

### Phase 4 — Validate

8. **Run the new Playwright tests** (always headed):
   ```bash
   cd e2e && E2E_LOCAL_DEPLOYMENT=true npx playwright test tests/{feature}/ --headed
   ```

9. **Fix any failures** — common issues:
   - **Timing**: Add `await expect(locator).toBeVisible()` before interacting
   - **Date pickers**: Use `click()` + `pressSequentially()` instead of `fill()` for spinbuttons
   - **Radio collisions**: Use `{ exact: true }` when labels are substrings of each other
   - **Multiple matches**: Scope with `.first()`, `.nth(n)`, or parent selectors like `getByRole('banner').getByText(...)`
   - **Step transitions**: Wait for next step's content to be visible before interacting

10. **Run again with slow-mo** if needed to debug timing:
    ```bash
    cd e2e && E2E_LOCAL_DEPLOYMENT=true npx playwright test tests/{feature}/ --headed --slow-mo=1000
    ```

### Phase 5 — Document

11. **Update test-report.md** — add a section noting which ACs now have deterministic tests:

    ```markdown
    ### Codified E2E Tests

    | AC | Test File | Test Name |
    |----|-----------|-----------|
    | AC-1 | e2e/tests/feature/create.spec.ts | can create an item through the form |
    | AC-2 | e2e/tests/feature/create.spec.ts | validation blocks submit without required fields |
    ```

## Page Object Conventions

- **File naming**: `{feature}.page.ts` in `e2e/pages/`
- **Locator prefix**: `l_` (e.g., `l_submit_button`, `l_description_textarea`)
- **Function prefix**: `f_` (e.g., `f_goto_create()`, `f_fill_date()`, `f_assert_on_show_page()`)
- **Register in barrel**: Export from `e2e/pages/index-pages.ts`
- **Use role-based selectors**: Prefer `getByRole()`, `getByPlaceholder()`, `getByText()` over CSS selectors
- **Exact matching**: Use `{ exact: true }` when a label is a substring of another (e.g., "Clinical" vs "Non-clinical")

## Common Patterns

### Date Picker (spinbutton segments)
```typescript
// Must use pressSequentially — fill() doesn't trigger Vue reactivity
const dayInput = page.getByRole('spinbutton', { name: 'day,' });
await dayInput.click();
await dayInput.pressSequentially('20');

const monthInput = page.getByRole('spinbutton', { name: 'month,' });
await monthInput.click();
await monthInput.pressSequentially('02');

const yearInput = page.getByRole('spinbutton', { name: 'year,' });
await yearInput.click();
await yearInput.pressSequentially('2026');
```

### Radio Group (exact matching)
```typescript
// Use exact: true to avoid "Clinical" matching "Non-clinical"
await page.getByRole('radio', { name: 'Clinical', exact: true }).click();
```

### Searchable Dropdown (CommonSelectMenu)
```typescript
await page.getByRole('button', { name: 'Show popup' }).first().click();
const option = page.getByRole('option', { name: 'John Recipient' });
await expect(option).toBeVisible();
await option.click();
```

### Step Wizard Transitions
```typescript
// Click Next
await page.getByRole('button', { name: 'Next' }).click();

// Wait for next step content before interacting
await expect(page.getByPlaceholder('Provide a detailed account')).toBeVisible();
```

### Page Navigation Assertions
```typescript
// URL pattern match
await expect(page).toHaveURL(/\/incidents\/\d+/);

// Element presence
await expect(page.getByRole('button', { name: 'Edit' })).toBeVisible();
```

### Scoping to Avoid Multiple Matches
```typescript
// If "John Recipient" appears in multiple places, scope to a region
await expect(page.getByRole('banner').getByText('John Recipient')).toBeVisible();
```

## Key Rules

- **Only codify PASSING ACs** — don't write tests for things that aren't working yet
- **Follow existing Playwright patterns** — check `e2e/tests/` and `e2e/pages/` for project conventions
- **One test per AC** — each acceptance criterion gets its own test case
- **Descriptive test names** — clearly describe the behaviour: `'can create an incident through the wizard'`
- **Always run headed** — use `--headed` flag so the user can see the browser
- **Always set E2E_LOCAL_DEPLOYMENT=true** — required for baseURL resolution from `.env`
- **Use slow-mo for debugging** — `--slow-mo=1000` when timing issues arise or user wants to follow along
- **Wait for Inertia** — `waitForLoadState('networkidle')` after navigation, `expect(locator).toBeVisible()` after step transitions
- **Use Page Objects** — keep locators and helpers in page objects, specs should read like user stories
