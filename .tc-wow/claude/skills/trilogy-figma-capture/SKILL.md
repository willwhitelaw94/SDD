# Trilogy Figma Capture Skill

Capture a live Portal page and push it to Figma as an editable design. Automates the full flow: browser navigation, authentication, Figma capture script injection, and polling for completion.

## When to Use

```bash
/trilogy-figma-capture <url>                        # Capture page, ask for output mode
/trilogy-figma-capture <url> --mode=new             # Create new Figma file
/trilogy-figma-capture <url> --mode=existing        # Add to existing Figma file
/trilogy-figma-capture <url> --mode=clipboard       # Copy to clipboard for pasting
/trilogy-figma-capture <url> --mode=toolbar         # Inject script only — use Figma toolbar manually
/trilogy-figma-capture --multi <url1> <url2> ...    # Capture multiple pages into one file
```

Run when you want to push a built Portal page into Figma for design review, comparison, or iteration.

## Prerequisites

| Tool | Required | Purpose |
|------|----------|---------|
| `mcp__figma__generate_figma_design` | Yes | Generate capture IDs and push to Figma |
| `mcp__chrome-devtools__navigate_page` | Yes | Navigate browser to target URL |
| `mcp__chrome-devtools__evaluate_script` | Yes | Inject capture script and trigger capture |
| `mcp__chrome-devtools__take_screenshot` | Yes | Verify page loaded correctly |
| `mcp__chrome-devtools__take_snapshot` | Recommended | Check for login page redirect |
| `mcp__chrome-devtools__click` | Recommended | Auto-login if needed |
| `mcp__chrome-devtools__wait_for` | Recommended | Wait for navigation after login |

## Execution Flow

### Step 0: Verify MCP Servers

Check that required MCP servers are available **before doing anything else**.

**Check Figma MCP:**
1. Use ToolSearch for `+figma generate` to verify `mcp__figma__generate_figma_design` is available
2. If **not available**, tell the user:
   ```bash
   claude mcp add figma --transport http https://mcp.figma.com/mcp
   ```
3. After install, user authenticates via browser (OAuth) on first use

**Check Chrome DevTools MCP:**
1. Use ToolSearch for `+chrome-devtools navigate` to verify `mcp__chrome-devtools__navigate_page` is available
2. If **not available**, tell the user:
   ```bash
   claude mcp add chrome-devtools -- npx -y chrome-devtools-mcp@latest
   ```

**If either MCP is missing** → ask user to install, restart Claude Code, and stop. The skill cannot proceed.

### Step 1: Parse Input & Ask Questions

**Parse the command arguments:**
- Extract URL(s) from the command
- Extract `--mode=` flag if provided
- Extract `--multi` flag if provided

**If no URL provided:** Ask the user which page to capture. Suggest using `mcp__laravel-boost__get-absolute-url` to build a valid URL from a route name.

**If no mode provided:** Ask the user:

```
Which output mode?
1. New Figma file (creates a fresh file in your team)
2. Existing Figma file (adds a page to an existing file)
3. Clipboard (copies design — paste into any open Figma file)
4. Toolbar (inject capture script only — use the Figma toolbar to manually choose what to capture)
```

**If existing file mode:** Ask the user for the Figma file URL and extract the `fileKey` from it (format: `figma.com/design/{fileKey}/...`).

### Step 2: Get Capture ID from Figma (newFile / existingFile only)

Based on the chosen output mode, call `mcp__figma__generate_figma_design`:

**New file:**
1. Call with `outputMode='newFile'` (no other params) → returns list of available teams/plans
2. Ask the user which team to use (or use the one they've used before)
3. Call again with `outputMode='newFile'`, `planKey=<selected>`, `fileName=<auto-generated from page title>`
4. Receive `captureId`

**Existing file:**
1. Call with `outputMode='existingFile'`, `fileKey=<extracted>`
2. Receive `captureId`

**Clipboard:**
- **Skip this step entirely.** Clipboard mode does NOT use a captureId. The Figma MCP returns capture instructions instead. Proceed directly to Step 3 (navigate) then Step 4 (inject + trigger with no captureId).

**Toolbar:**
- **Still get a captureId** — call `mcp__figma__generate_figma_design` with the desired output mode (newFile or existingFile) to get a captureId. The toolbar needs hash params containing the captureId to know where to send the capture.
- If the user hasn't specified a destination, call without `outputMode` first to get options, then ask the user.

**Save the `captureId` (newFile/existingFile only) — you'll need it for capture and polling.**

### Step 3: Navigate & Authenticate

1. **Navigate** to the target URL:
   ```
   mcp__chrome-devtools__navigate_page(
     type: 'url',
     url: <target-url>
   )
   ```

2. **Check if redirected to login** — take a snapshot:
   ```
   mcp__chrome-devtools__take_snapshot()
   ```
   Look for login page indicators: page title contains "Log in", URL contains `/login`, or presence of login form elements.

3. **If login page detected** — auto-login (local dev only):
   a. Find and click the "Show popup" button (DevLogins dropdown) using `mcp__chrome-devtools__click`
   b. Select "Admin" from the listbox options using `mcp__chrome-devtools__click`
   c. Find and click the "Log in" button using `mcp__chrome-devtools__click`
   d. Wait for navigation using `mcp__chrome-devtools__wait_for` with expected text from the target page (e.g., the page title)
   e. If not on the target URL, navigate to it again

4. **Take a screenshot** to confirm the correct page loaded:
   ```
   mcp__chrome-devtools__take_screenshot()
   ```
   Show the screenshot to the user for visual confirmation before proceeding.

### Step 4: Inject Capture Script & Trigger

Inject the Figma capture script via `mcp__chrome-devtools__evaluate_script` in **two separate calls** for reliability:

**Call 1 — Inject the script:**
```javascript
async () => {
  const r = await fetch('https://mcp.figma.com/mcp/html-to-design/capture.js');
  const code = await r.text();
  eval(code);
  await new Promise(resolve => setTimeout(resolve, 2000));
  return typeof window.figma !== 'undefined' ? 'ready' : 'failed';
}
```

**Call 2 — Trigger the capture:**

For **clipboard mode** (no captureId):
```javascript
async () => {
  const result = await window.figma.captureForDesign({ selector: 'body' });
  return result;
}
```
This copies the design to the clipboard directly. A "Copied to clipboard" toast will appear. No polling needed.

For **newFile / existingFile** (with captureId):
```javascript
async () => {
  return await window.figma.captureForDesign({
    captureId: '<CAPTURE_ID>',
    endpoint: 'https://mcp.figma.com/mcp/capture/<CAPTURE_ID>/submit',
    selector: 'body'
  });
}
```
Replace `<CAPTURE_ID>` with the actual captureId from Step 2.

For **toolbar mode** (manual capture via default browser):

**IMPORTANT:** The toolbar does NOT work in Chrome DevTools MCP's headless browser — hash params trigger the capture script to blank the page. Toolbar mode requires the user's default browser.

1. **Temporarily add the Figma capture script to `app.blade.php`:**
   Add `<script src="https://mcp.figma.com/mcp/html-to-design/capture.js" async></script>` to the `<head>` section, just before the `<!-- Scripts -->` comment.

2. **Open the URL in the user's default browser** with hash params:
   ```bash
   open "<target-url>#figmacapture=<CAPTURE_ID>&figmaendpoint=https%3A%2F%2Fmcp.figma.com%2Fmcp%2Fcapture%2F<CAPTURE_ID>%2Fsubmit&figmadelay=1000&figmaselector=*"
   ```
   - `figmaselector=*` enables manual element selection UI
   - `figmadelay=1000` waits 1s before showing toolbar (lets page render)
   - The script must be in the HTML **before** page load — dynamically injecting it after load does NOT trigger the toolbar

3. **Tell the user:** "Figma capture toolbar is now active in your browser. Use it to capture."

4. **After user confirms capture is done**, remove the temporary script tag from `app.blade.php`.

5. **Poll for completion** using `mcp__figma__generate_figma_design(captureId=<CAPTURE_ID>)` — the toolbar submits the capture server-side, so polling still works.

**Why this approach:** The capture script checks for `#figmacapture=` hash params on initial page load. If found, it renders the toolbar UI. This only works when the script is embedded in the HTML source before the page loads — not when injected dynamically via `evaluate_script`.

**Important:** Do NOT use the `#figmacapture` hash approach through Chrome DevTools MCP — it blanks the page. For automated (non-toolbar) capture, always use the two-call `evaluate_script` approach above.

**If the evaluate_script times out:** This is expected for large pages — the capture may still have submitted successfully. Proceed to polling (newFile/existingFile) or take a screenshot to check for the toast (clipboard).

### Step 5: Poll for Completion (newFile / existingFile only)

**Skip this step for clipboard mode** — clipboard captures complete immediately when `captureForDesign` returns `{success: true}`.

**For toolbar mode:** Wait for the user to confirm they've used the toolbar, then poll with the captureId to verify the capture completed.

For newFile/existingFile, call `mcp__figma__generate_figma_design(captureId=<CAPTURE_ID>)` to check status.

- **If pending:** Wait 5 seconds and poll again (up to 5 retries)
- **If completed:** Extract the Figma URL from the response

### Step 6: Return Result

**New file:**
1. Open the claim URL in the browser: `open "<claim-url>"`
2. Tell the user: "Your Figma file is ready: <url>"
3. Save the `fileKey` for potential multi-page follow-up

**Existing file:**
1. Tell the user: "Page added to your Figma file"
2. Provide the file URL

**Clipboard:**
1. Tell the user: "Design copied to clipboard — open Figma and paste (Cmd+V)"

**Toolbar:**
1. Tell the user: "Figma capture toolbar is active in your default browser. Use the toolbar buttons to capture and send to Figma."
2. When user confirms capture is done, remove the temporary script tag from `app.blade.php`.
3. Poll with `captureId` to confirm completion (same as newFile/existingFile).

### Step 7: Multi-Page Capture (Optional)

If using `--multi` flag or user asks to capture more pages:

1. After the first capture completes, extract the `fileKey` from the response
2. For each additional URL:
   a. Call `mcp__figma__generate_figma_design(outputMode='existingFile', fileKey=<fileKey>)` to get a new captureId
   b. Navigate to the URL (auth session persists)
   c. Inject capture script and trigger (Step 4)
   d. Poll for completion (Step 5)
3. After all pages captured, open the Figma file for the user

**Important:** The first capture must complete before starting additional captures (it creates the file). Subsequent captures CAN run in parallel since the file already exists, but for reliability run them sequentially.

## Behavior Rules

1. **Always verify MCP servers first** — don't attempt capture without both Figma and Chrome DevTools MCP
2. **Always authenticate automatically** — use the DevLogins dropdown pattern from `trilogy-pixel-perfect`
3. **Always show a screenshot before capturing** — let the user confirm the right page is loaded
4. **Handle timeouts gracefully** — `evaluate_script` may time out for large pages; poll anyway
5. **Save fileKey for multi-page** — enable follow-up captures to the same file
6. **Auto-generate file names** — use the page title or URL path for the Figma file name
7. **Kill stale Chrome processes if needed** — if Chrome DevTools MCP fails with "browser already running", kill processes and remove SingletonLock:
   ```bash
   pkill -f "chrome.*chrome-devtools-mcp"
   rm -f ~/.cache/chrome-devtools-mcp/chrome-profile/SingletonLock
   ```
   Then retry navigation.
