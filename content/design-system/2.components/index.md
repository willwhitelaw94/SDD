# Components

Components are the building blocks of the TC Portal UI. Each component is documented with its Figma source, available variants, and implementation details.

- [Accordion](/design-system/components/accordion) — Collapsible content sections for progressive disclosure
- [Badges](/design-system/components/badges) — Badge variants, sizes, colours, and specialized badge components
- [Buttons](/design-system/components/buttons) — Button variants, sizes, states, and icon buttons
- [Forms](/design-system/components/forms) — Input components, layout patterns, and spacing
- [Modals & Dialogs](/design-system/components/modals-and-dialogs) — Modal overlays, confirmation dialogs, and the composable dialog system
- [Cards](/design-system/components/cards) — General cards, action cards, KPI cards, and empty states
- [Tables](/design-system/components/tables) — Data tables with sorting, filtering, pagination, and actions
- [Navigation](/design-system/components/navigation) — Sidebar navigation patterns, component architecture, and interaction states
- [Stepper](/design-system/components/stepper) — Multi-step navigation for portal forms and onboarding flows
- [Tabs](/design-system/components/tabs) — Tab navigation sizes, states, and usage patterns
- [Avatars](/design-system/components/avatars) — Avatar sizes, colours, initials fallback, and avatar groups
- [Dropdown Menu](/design-system/components/dropdown) — Context menus, grouped items, and custom triggers
- [Data Display](/design-system/components/data-display) — Progress bars, spend bars, doughnut charts, and legends
- [Interactive](/design-system/components/interactive) — Command palette, carousel, and pin input
- [Layout](/design-system/components/layout) — Page headers, definition lists, and definition items
- [Feedback & States](/design-system/components/feedback) — Alert banners, empty placeholders, and system feedback
