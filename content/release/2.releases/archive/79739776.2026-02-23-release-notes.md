## TL;DR

This is one of the most significant releases in recent months, shipping three major new capabilities alongside a substantial developer experience improvement. Teams can now be assigned fine-grained permissions independently from individual user roles, giving organisations far more control over who can do what. The mobile app now has a full API foundation — authentication, Services Australia entry and departure management, and access to recipient bills and packages are all live. Weekly care management bills for RC and EL packages are now created automatically every Monday, replacing a fully manual process. Underpinning all of this is a comprehensive TypeScript safety overhaul across the Vue component library, making the frontend more robust and developer-friendly.

### Fine-Grained Team Permissions

Organisations can now control exactly which actions each internal team is permitted to perform — independently of individual staff roles.

**Why this matters**

Role-based access was too broad for many organisations. A staff member's role granted access to everything that role allows, even when that capability wasn't relevant to their team's remit. Teams needed a way to further restrict what their members can do without changing everyone's individual role. This closes a compliance and governance gap.

**What's changing**

A new permission management layer has been introduced at the team level. Each team can be configured with a specific set of permissions, and those permissions now take precedence when determining what a team member can do within the context of their team. Team management itself also gained full CRUD capabilities — teams can be created, renamed, soft-deleted, and have their members managed.

**Who's affected**

Admin staff responsible for managing internal teams and access control. Any staff member whose permissions may now be further restricted by their team's settings.

**Action required**

This feature is being rolled out as part of a phased activation. Teams will be configured individually. Administrators should engage with their implementation contact to discuss how permissions should be structured for their organisation.

### Automated Weekly Care Management Bills

RC and EL care management bills are now created automatically every Monday, eliminating a manual billing creation step.

**Why this matters**

Creating weekly care management bills was a manual, time-consuming task prone to omission. Automating this removes operational overhead and ensures billing consistency regardless of staff availability.

**What's changing**

A scheduled action runs every Monday morning to generate RC/EL care management bills. The previous manual process is no longer required for standard weekly cycles.

**Who's affected**

Finance teams and billing staff who previously managed this manually.

**Action required**

No action required for the automated process. Finance teams should review generated bills to confirm accuracy in the first few cycles after go-live.

---

### Mobile App API Foundation

The portal now exposes a dedicated API for the Trilogy Care mobile application, covering authentication, Services Australia entry and departure management, and recipient bill access.

**Why this matters**

The mobile app needed a stable, secure API to retrieve recipient data and interact with Services Australia services. This release provides the full backend foundation required for mobile app functionality.

**What's changing**

Recipients can now authenticate via the mobile app and access their package information, billing history, and Services Australia service events. Internal staff can manage entry and departure events for Support at Home services via the API.

**Who's affected**

Recipients using or planning to use the Trilogy Care mobile app. Staff involved in Services Australia data management.

**Action required**

No action required for the portal web application. Mobile app availability will be communicated separately.

---

### Bulk Import of Care Management Bills via CSV

Finance administrators can now upload a CSV file to bulk-create care management bills, replacing one-by-one manual entry.

**Why this matters**

Processing large volumes of care management bills individually is slow and error-prone. CSV import gives finance teams a fast, auditable way to create many bills at once.

**What's changing**

A new Nova admin action accepts a CSV file containing care management bill data. The import validates each row, reports invalid entries, and creates bills for all valid rows. Results are logged including counts of valid rows, invalid rows, bills created, and any errors.

**Who's affected**

Finance teams and administrators with access to the Nova admin panel.

**Action required**

No action required — the import tool is available immediately to authorised administrators in Nova.

---

### Budget Plan Creation API

The budget management system now supports creating budget plans via API, enabling the next phase of budget workflow automation.

**Why this matters**

Budget plan creation was constrained to manual flows. This API endpoint enables automation, integration with external tools, and the groundwork for a mobile-accessible budget management experience.

**What's changing**

A new endpoint allows budget plans to be created programmatically. The system validates commencement quarter, budget limits, and contribution rates before accepting a new plan.

**Who's affected**

Development and integration teams building on top of the portal. Ultimately, staff and recipients who benefit from automated budget workflows.

**Action required**

No immediate action required. This is infrastructure for upcoming capability delivery.

### Team Permissions Configuration

A new team-level permission system has been introduced, allowing organisations to assign specific permissions to teams independently of user roles.

**Why this was changed**

Organisations needed more granular access control than individual roles provided. Teams can now be restricted to only the permissions relevant to their function.

**What's different**

Previously, a user's role determined all their permissions. Now, their team's configured permissions create an additional layer of access control. Team management itself is now available via a dedicated admin interface with full create, edit, delete and member management capability.

**Where in the platform**

`Staff → Teams` — for team management and member assignment. Permission configuration is handled by administrators via the Teams admin interface.

**How to help users**

Walk users through their team's settings. If a user reports they can't access something they previously could, it may be because their team's permissions have been configured more narrowly. Check their team membership and the team's permission set.

**Likely Q&A**

> **Q: "I can't access something I used to be able to do?"**
> A: Their team's permissions may now restrict that action. Check which team they're in and what permissions that team has. If it should be available, an admin can adjust the team's permission set.

> **Q: "I don't see the Teams section in the menu?"**
> A: This feature is being rolled out per organisation. If their organisation hasn't had it activated yet, the Teams section may not be visible.

> **Q: "A user was removed from a team but still has access?"**
> A: Permission contexts are refreshed via middleware on each request, but ask the user to log out and back in to be certain their session is fully refreshed.

Permissions: Requires `Admin` role to manage teams and permissions.

Manual weekly bill creation was time-consuming and inconsistent. Automation ensures bills are always generated on schedule.

### Care Management Bill CSV Import (Nova)

Administrators can now bulk-import care management bills via CSV upload in Nova.

**Why this was changed**

Creating large volumes of care management bills individually was inefficient and error-prone.

**What's different**

A new Nova action `CareManagementBillImport` accepts CSV files. Results include counts of valid rows, invalid rows, bills created, and errors. A dry-run mode is available to preview results before committing.

**Where in the platform**

Nova admin panel → Care Management Bill Imports resource.

**How to help users**

If an import fails validation, direct the user to review the error column in the results output. Each invalid row will have an explanation.

**Likely Q&A**

> **Q: "Some rows weren't imported — why?"**
> A: The import validates each row. Invalid rows are skipped and logged. The user should check the `results` field in the import record.

> **Q: "Can I undo an import?"**
> A: Not automatically. Escalate to the dev team if a large incorrect import needs reversal.
