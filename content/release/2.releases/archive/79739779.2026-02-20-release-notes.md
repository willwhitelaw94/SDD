## TL;DR

This release introduces team-based access control, giving organisations a structured way to group staff and scope their permissions. It also delivers automated weekly care management bill processing for Support at Home packages, a new mobile application authentication API, and the groundwork for a V2 Budget API. Alongside these major additions, the release includes improved funding allocation logic, a TC/CC fee exemption capability on bills, Services Australia Entry and Departure API integration, and dozens of targeted bug fixes and quality improvements across billing, supplier management, and document handling.

## What's New?

::tabs{default="Business View"}
:::tab{label="Business View"}

### Team-Based Access Control

Organisations can now create named teams with defined permission boundaries, giving managers precise control over who can do what across different parts of the platform.

**Why this matters**

Previously, access was managed at the individual role level with no way to group staff into operating units. As organisations grow, managing permissions individually becomes error-prone and time-consuming. Teams provide a structural layer that mirrors how organisations actually work.

**What's changing**

Staff can be organised into teams, each with their own permission set. Team leaders can manage membership while Admins control what each team is allowed to do across the portal.

**Who's affected**

Administrators and managers responsible for staff access. Compliance and operations teams who need scoped access to specific modules.

**Action required**

Admins will need to create teams and assign members once deployed. No existing permissions or roles are removed — this is an additive capability.

---

### Automated Care Management Billing

Weekly care management bills for Support at Home packages with End of Life (EL) and Restorative Care (RC) classifications are now generated automatically every Monday, with a Nova import tool for bulk CSV creation when needed.

**Why this matters**

Manual care management bill creation was consuming significant Finance team time each week and introduced risk of missed or inconsistent billing. Automating this process removes the overhead and improves billing accuracy.

**What's changing**

The system will automatically create the correct care management bills each Monday. Finance staff can also import care management bills in bulk via a CSV file in the Nova admin panel.

**Who's affected**

Finance and billing teams responsible for Support at Home package billing. Care management administrators.

**Action required**

Review and test the automation in staging before production release. Confirm the `CM_WEEKLY_BILLING_ENABLED` environment variable is set correctly per environment.

---

### Mobile Application Authentication

A dedicated authentication API for the Trilogy Care mobile application is now available, supporting login, logout, and password management. CORS settings have been updated to allow local development and mobile testing environments.

**Why this matters**

The mobile application required a secure, dedicated authentication pathway separate from the web portal. This is a foundational step in the mobile product launch.

**What's changing**

New API endpoints at `/api/v1/auth/` handle mobile authentication. This is currently in rollout phase and is not user-facing in the web portal.

**Who's affected**

Mobile application users and the team building the mobile product.

**Action required**

No action required for existing portal users. Mobile team to verify authentication flows in staging.

---

### Budget V2: Funding Allocation and Budget API

A new API endpoint for creating budget plans is now available, with updated funding allocation logic that prioritises Restorative Care (RC) funding and handles the distinction between planned and unplanned services more accurately.

**Why this matters**

The existing budget system did not reflect the correct funding stream hierarchy for Support at Home packages, leading to incorrect billing. The new allocation logic ensures RC funding is used first when available and that duplicate unplanned services cannot be added.

**What's changing**

Funding allocations on budget items now show streams in their configured priority order rather than through a manual dropdown. Restorative Care is used first when available. The V2 Budget API enables external integrations and the mobile app to create budget plans programmatically.

**Who's affected**

Care Coordinators and Finance staff working with Support at Home budgets.

**Action required**

No action required — existing budgets are unaffected. New budget creation via API requires the V2 endpoint.

---

### TC/CC Fee Exemption on Bills

A new dropdown control on the Bill page allows eligible organisations to apply a TC (Trilogy Care) or CC (Care Coordinator) fee exemption, giving Finance teams more flexibility in fee management for VC-approved packages.

**Why this matters**

Some packages qualify for fee exemptions that were previously not manageable directly in the portal, requiring workarounds. This brings that capability into the platform.

**What's changing**

A TC/CC fee exemption dropdown appears next to the VC Approved toggle on the Bill page. Only Admin users can see all exemption options. SM-level packages only show TC exemption.

**Who's affected**

Admin users and Finance teams managing bills with VC-approved packages.

**Action required**

No action required — the feature is available immediately to Admin users. Confirm role assignments if users report they cannot see the control.

---

### Business Verification Document Protection

Business verification documents can no longer be deleted, only replaced. This prevents accidental removal of legally significant documents during the supplier onboarding and compliance review process.

**Why this matters**

Accidental deletion of verification documents was causing compliance gaps and support overhead. The replace-only workflow preserves document history while still allowing updates.

**What's changing**

The delete action is removed for business verification documents. A replace workflow is available instead.

**Who's affected**

Compliance teams and staff managing supplier onboarding documentation.

**Action required**

No action required — the behaviour changes automatically.

---

### Services Australia Entry & Departure API

New proxy endpoints for the Services Australia Entry and Departure APIs are now available, enabling entry and discharge lifecycle events to be submitted directly from the portal.

**Why this matters**

Entry and departure submissions to Services Australia are required for regulatory compliance. Having these integrated in the portal reduces manual handling and the risk of missed submissions.

**What's changing**

New CRUD API endpoints follow the same pattern as existing Services Australia integrations (Claim, Invoice, Budget, Payment Statement).

**Who's affected**

Finance and compliance teams responsible for Services Australia submissions.

**Action required**

Requires feature enablement and configuration per environment. Integration testing required before use.

---

### Compliance Team Member: Organisation Management Access

The Compliance Team Member role now has access to organisation management permissions, allowing compliance staff to perform supplier management tasks without requiring an Admin or Manager role.

**Who's affected**

Compliance Team Members who need to manage supplier details or replace documents.

**Action required**

No action required — the permission is applied automatically to the Compliance Team Member role.

### You Can Now Organise Staff Into Teams

Administrators can create named teams, assign members, and control what each team is allowed to do — giving your organisation a cleaner way to manage who has access to what.

**Why we built this**

As organisations grow, managing individual roles becomes complicated. Teams let you group people the way your organisation actually operates, and control their access at the team level.

**What's changed**

A new Teams section is available in the Staff area. You can create teams, add members, set a team leader, and assign permissions to the team. Activity changes (adding/removing members, renaming) are recorded in the team timeline.

**Where to find it**

`Staff → Teams`

**How to use it**

1. Go to `Staff → Teams`
2. Click **Create Team**
3. Enter a team name and select a team leader
4. Add members by searching for staff users
5. Assign permissions to the team as needed
6. Save the team

### TC/CC Fee Exemption on Bills (Admin Only)

Admin users can now apply a TC or CC fee exemption directly on the Bill page, removing those fees for eligible packages without requiring any manual workarounds.

**Why we built this**

Fee exemptions were previously not manageable in the portal, requiring finance staff to apply workarounds. This brings the control directly into the billing workflow.

**What's changed**

A fee exemption dropdown now appears next to the VC Approved toggle on the Bill page. Options shown depend on your role and the package level.

**Where to find it**

`Bills → [Bill] → Fee Exemption dropdown`

**How to use it**

1. Open a Bill
2. Locate the **Fee Exemption** dropdown near the VC Approved toggle
3. Select the appropriate exemption (TC, CC, or none)
4. The change is recorded in the bill history

You'll only see this control if you have Admin access. For SM packages, only the TC exemption option is available.

### Team Permissions & Management MVP

**Why this was changed**

Organisations requested a way to group staff into teams with scoped permissions rather than managing individual roles in isolation. This is the first version of that capability.

**What's different**

A new **Teams** section under Staff allows Admins to create teams, add members, assign team leaders, and grant permissions at the team level. Activity is logged in a team timeline.

**Where in the platform**

`Staff → Teams` (Admin access required to create/manage teams)

**How to help users**

Team creation requires Admin access. Team leaders can manage membership but cannot modify permissions.

**Likely Q&A**

> **Q: "I can't see the Teams section?"**
> A: Teams is only visible to users with Admin access. Ask their Admin to check their role assignment.

> **Q: "A team member can't see a feature they should have access to?"**
> A: Check whether the team has the relevant permission assigned via `Staff → Teams → [Team] → Permissions`.

> **Q: "Why does the team show historical members that have been removed?"**
> A: The activity timeline records all additions/removals for audit purposes. Removed members are not active — this is expected behaviour.

### Automated Care Management Billing

**Why this was changed**

Finance teams were creating care management bills manually each week for SaH packages with EL and RC classifications. Automation reduces missed billing and manual effort.

**What's different**

Weekly care management bills for eligible SaH packages are now generated automatically every Monday. A Nova CSV import tool also allows bulk creation.

**Where in the platform**

Nova Admin Panel: Care Management Bill Imports (for CSV import); automated billing runs as a scheduled job.

**How to help users**

If a user reports a care management bill was not created, check whether the package has EL or RC classification and whether the automation was enabled for that environment.

**Likely Q&A**

> **Q: "Why does the bill show as 'on hold'?"**
> A: Automated bills are placed on hold when a review validation issue is detected. The on-hold reason will describe the issue. Finance staff need to review and release them.

> **Q: "The CSV import didn't create all expected bills?"**
> A: Check the import results in Nova — it records valid rows, invalid rows, and any errors. Invalid rows may have failed validation.
