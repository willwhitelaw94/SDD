Design Principles
The strategic guidelines driving every design decision
North Star
Speed over polish
Optimise for agent efficiency. Fastest possible path to update a lead.
Principle 1
Context without navigation
Key lead info (stage, status, assignment, last contact) visible from any tab without clicking back.
Principle 2
Edit where you read
Section cards toggle between read and edit mode in-place. No modals, no page navigation.
Principle 3
Pipeline first
Journey Stage and Lead Status are the most prominent elements on every screen. They're what the business measures.
Principle 4
Familiar > clever
Use patterns sales staff already know from Zoho/CRMs. Don't innovate on interaction patterns, innovate on speed.
Frustration Risks
⚠

Too many clicks to update a single field (Zoho complaint)

⚠

⚠

Not knowing what's required before a stage transition (wizard should preview requirements)

⚠

Timeline buried under too much scrolling (paginate, filter)

Competitive Analysis
Learnings from 5 CRM platforms — click to expand details
H

HubSpot CRM

3-column contact record

What they do well

  • Customisable overview tab with drag-and-drop cards
  • Activity timeline filterable by type (calls, emails, notes, tasks)
  • Sidebar properties grouped into collapsible cards

What we learned

  • Overview tab as default landing — quick summary before diving in
  • Sidebar should show identity + status at a glance without scrolling
S

Salesforce

3-column with 4-zone sidebar

What they do well

  • Path component — horizontal stage bar with click-to-advance + guidance
  • Collapsible sections with remembered preferences
  • Activity timeline with expand/collapse all

What we learned

  • Stage guidance text per step (tell agent what's needed)
  • Zone-based field prioritisation
  • 3-column overkill for our use case
P

Pipedrive

2-panel with independent scroll

What they do well

  • Summary panel at top — critical info always visible
  • Independent scrollbars for sidebar and main content
  • Focus section highlights upcoming activities, pinned notes

What we learned

  • Independent scrolling is essential
  • Summary panel pattern for persistent sidebar
C

Close CRM

2-panel with pinned activities

What they do well

  • Pinned activities (up to 5) float to top of timeline
  • Unified activity stream — all comm types in one feed
  • Auto-logging eliminates manual data entry
  • Inline editing with ESC to cancel

What we learned

  • Pinned notes pattern for timeline
  • Activity filtering by type without leaving page
  • Speed-first philosophy matches our principle
Z

Zoho CRM

The system we're replacing — tabs + related lists

What they do well

  • Blueprint workflow — visual stage progression
  • Canvas view for custom visual layouts
  • Related lists with expand/collapse

Why users want to leave

  • Timeline defaults to oldest-first, no persistent preference
  • Interface complexity — steep learning curve
  • Missing context on updates
  • Performance degrades with many related lists

What we must improve

  • Timeline newest-first by default
  • Simpler, consistent interface
  • Clear audit trail showing who/when/why on every change

Stage Visualisation

Horizontal progress bar at page top

☰

Timeline Position

Dedicated tab (not always-visible panel)
Keeps Overview tab clean. Staff go to Timeline when they need history.
✎

Section Editing

Inline card expansion (ACRM pattern)
Spec decision. Collapsible cards with edit toggle — read mode by default.
☰

Sidebar Content

Identity + status + assignment (persistent)
Pipedrive/Close pattern — key context always visible regardless of tab.
📌

Notes

Pinned notes support (Close pattern)
High-value for lead handoffs between agents.

Edge Cases
9 edge cases identified — filter by priority
P1 Critical

Duplicate email on create

Warning banner with link to existing lead. Allow proceed.

P1 Critical

Lead with no Journey Stage data

Default to Pre-Care on creation. Show "Unknown" badge for legacy leads.

P1 Critical

Lost lead re-engagement

Allow status change back from Lost. Require re-engagement note. Stage unfreezes.

P2 Important

Empty timeline

Show empty state with "No activity yet" message.

P2 Important

Missing attribution data

Show "No attribution data" empty state on Other tab.

P2 Important

Bulk assign 100+ leads

Paginated selection. Background job for bulk assignment. Success toast.

P2 Important

Zoho sync conflict

Portal wins for fields edited in Portal. Show "Last synced" timestamp.

P3 Nice-to-have

Long lead name (overflow)

Truncate with ellipsis in list view. Full name in profile sidebar.

P3 Nice-to-have

Consumer with no account (guest)

Section cards still display. "Create account to save" prompt for guests.

Assumptions

2Timeline tab is sufficient — staff don't need always-visible activity feed
36 primary Journey Stages are enough — substages provide granularity without overwhelming the progress bar
4Consumer section-based forms will increase completion vs the current 6-step wizard
