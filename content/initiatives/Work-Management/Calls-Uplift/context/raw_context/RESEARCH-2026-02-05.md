## Executive Summary

TC Portal has a **production-ready Aircall integration** used by 250+ care coordinators. The system captures calls, stores recordings in S3, and links them to packages. However, **66% of calls remain untagged** due to manual tagging friction - coordinators must manually add TC customer numbers to Aircall comments.

**Key opportunity:** With Trilogy Graph providing centralized data, we can implement **phone number auto-matching** without relying on manual tagging. The existing codebase already has phone normalization infrastructure (`HasPhoneNumber` trait, E.164 formatting) that can be extended.

## Technical Architecture

### Database Schema
```
aircall_calls
├── id, aircall_id, contact_id, user_id
├── direction (inbound/outbound)
├── status (completed/failed/no-answer)
├── started_at, answered_at, ended_at, duration
├── raw_digits (caller/callee number - NOT normalized)
├── recording (Aircall URL), s3_recording_path
├── missed_call_reason
└── meta (JSON)

aircall_aircallables (polymorphic pivot)
├── aircall_call_id
├── aircallable_id, aircallable_type (Package, User)
└── Unique constraint
```

### Webhook Processing Chain
```
Aircall Event → Webhook → ProcessCallWebhookJob → [Chain]:
1. CreateAircallCallFromCallJob (create/update record)
2. StreamAndStoreCallRecordingToS3 (store recording)
3. AttachUserToCallJob (link Aircall user)
4. AttachUserFromContactJob (link contact as User by phone/email)
5. AttachPackageFromCommentsJob (TC# matching from comments)
```

### Phone Number Data Sources
| Source | Location | Format | Notes |
|--------|----------|--------|-------|
| Package Recipient | `users.phone_e164` | E.164 | Via User model |
| Care Circle Contacts | `package_representatives.meta['phone_e164']` | E.164 | HasPhoneNumber trait |
| Aircall Participant | `Participant.phoneNumber` | Raw | From webhook payload |
| Call raw_digits | `aircall_calls.raw_digits` | Raw | NOT normalized |

### Key Files
- `app-modules/aircall/` - Core Aircall module
- `app/Jobs/Aircall/Webhooks/Call/AttachPackageFromCommentsJob.php` - Current linking logic
- `app/Jobs/Aircall/Webhooks/Call/AttachUserFromContactJob.php` - Phone matching pattern (reusable)
- `resources/js/Pages/Packages/tabs/PackageComms.vue` - Frontend component
- `resources/js/Components/Package/AircallCallsTable.vue` - Call table display

## Trilogy Graph Opportunity

**Note from stakeholder:** Aircall data will come through Trilogy Graph.

This changes the technical approach:

### Before (Direct Aircall Improvement)
- Enhance webhook processing
- Add phone number normalization at ingest
- Create new matching jobs

### After (Trilogy Graph Integration)
- Trilogy Graph provides centralized phone number data
- Graph can match calls to packages using relationships
- Portal queries Graph for call-to-package associations
- Reduces need for complex webhook processing

### Implications
1. **Phone matching logic moves to Graph** - Not TC Portal's responsibility
2. **Call Bridge UI becomes primary focus** - UX for coordinators during calls
3. **Activity capture** - Still needed in Portal for care management tracking
4. **Recording access** - Portal still manages S3 playback

## Proposed Solution: Call Bridge UI

### What It Is
An in-app interface that gives care coordinators context when on calls:
- See package details while on a call
- Quick-link calls to packages (if Graph auto-match fails)
- Log call as care management activity
- Add notes during/after call
- Access call history for the contact

### Why It Matters
- Reduces friction for tagging (no need to go to Aircall app)
- Provides work management context (tasks, notes, timeline)
- Enables automatic activity capture
- Improves data quality (more calls properly attributed)

### MVP Features
1. **Call notification banner** - Shows when coordinator is on a call (via Aircall status)
2. **Package context panel** - If match found, show package details
3. **Manual link button** - If no match, allow quick search and link
4. **Activity log checkbox** - "Log as care management activity"
5. **Quick note field** - Add call summary
